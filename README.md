# NFT Mobile App Automation

A Katalon Studio project for automating functional and regression testing of a mobile NFT application.

## Overview

This repository contains reusable Katalon test assets, project configuration, custom Groovy keywords, execution profiles, and self-healing test-object data. It is intended to provide a maintainable foundation for local and CI-based mobile test execution.

## Technology

- **Katalon Studio** 5.9.0 or later
- **Groovy** for custom keywords and test automation logic
- **Generic Katalon project** configuration
- **Selenium Remote WebDriver** support through local, cloud, or device-farm configuration

## Repository structure

| Path | Description |
| --- | --- |
| `katalon_NFT_project.prj` | Katalon project metadata and source-folder configuration |
| `Include/config/` | Logging and test configuration |
| `Libs/` | Custom keywords and generated global-variable support |
| `Profiles/` | Katalon execution profiles |
| `Reports/Self-healing/` | Self-healing test-object data generated by Katalon |
| `settings/` | Katalon execution and integration settings |
| `console.properties.example` | Safe template for local or remote execution properties |

## Prerequisites

1. Install Katalon Studio with the required mobile execution support.
2. Install and configure the required Android or iOS tooling if tests run on a local device or emulator.
3. For remote execution, obtain the required device-farm or Selenium Grid details from your team.
4. Confirm that the application build and test data are available in the target environment.

## Configuration

Sensitive and environment-specific values must remain local and must not be committed to GitHub.

1. Copy `console.properties.example` to `console.properties`.
2. Add the required remote WebDriver, device, and integration values locally.
3. Select or create the appropriate Katalon execution profile in `Profiles/`.
4. Store credentials in a secure secret manager or CI/CD secret store; do not place them in source files, screenshots, reports, or commit messages.

`console.properties` is excluded from version control by `.gitignore`. If a credential has ever been committed, revoke or rotate it immediately and remove it from the repository history before publishing.

## Running tests

1. Open the project in Katalon Studio.
2. Configure the target device or remote execution endpoint.
3. Select the required execution profile.
4. Run the relevant test case or test suite.
5. Review the generated execution results and self-healing information locally.

The exact test-case and test-suite paths depend on the assets included in the project and the test scope configured by the automation team.

## Development guidelines

- Keep reusable behavior in `Libs/CustomKeywords.groovy`.
- Keep environment-specific configuration outside tracked source files.
- Use descriptive names for test cases, test objects, and execution profiles.
- Do not commit credentials, personal data, internal URLs, device identifiers, or generated reports containing sensitive information.
- Review `git diff --cached` before every push.

## CI/CD integration

The project can be integrated with a CI/CD pipeline by supplying configuration through protected environment variables or secret files generated at runtime. The pipeline should:

- Install or invoke the supported Katalon execution environment.
- Inject secrets only at runtime.
- Execute the selected test suite or test collection.
- Publish test results as protected build artifacts.
- Avoid exposing secrets in command output and logs.

## Security

This repository is a sanitized project template. Before making it public, scan the complete Git history—not only the current working tree—for credentials and private data. Rotate any exposed credential and use a history-rewriting tool when necessary.
